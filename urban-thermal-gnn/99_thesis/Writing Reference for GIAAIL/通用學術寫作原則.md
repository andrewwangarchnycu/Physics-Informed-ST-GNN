# 通用學術論文寫作原則（可移植）

適用：繁體中文（台灣學術慣用）碩博士論文與期刊投稿。與特定研究主題、方法、領域無關；供任何論文專案的撰寫／精修／審查 agent 使用。標記「〔專案風格〕」者為可依各專案設定調整的樣式選擇，非普遍規範。

## §0 最高頻鐵則

1. 段首句＝可被反駁的論點或約束／衝突，不是中立觀察、不是元件宣告、不是寫作方法自述。
2. 禁說教／後設寫作：「正確的第一步是…」「本節將…」「須先言明的是」「讀圖時可循…」。直接做，不描述在做。
3. 因為→所以的論證鏈，不要「而且→然後」流水帳。動態因果（為解決 X→引入 Y／此限制迫使 Z），非靜態報告（負責／包含／由…組成）。
4. 一概念一 owner：每個論點只在擁有它的小節寫完整，他處以名詞帶過。跨節不重述，指回已讀前節的參照（backward reference）一律刪。
5. 用「本研究提出／建立／驗證」，不用「試圖／希望／期望」；謙遜詞每段最多一次。
6. 未驗證不用「證明／顯著」，改「顯示／表明／呈現／推估／傾向」。斷言先於證據者凍結為條件式（明文限定適用範圍）。
7. 術語全文一致，同一概念固定一詞。首見 `中文全稱（English Full Name, ABBR）`，之後只用縮寫，不再重複雙語。
8. 大陸用語定稿前全掃（§4.2）。
9. 數值後必有解讀句（「…顯示…」）；圖表必先被正文引用並解讀，無孤兒圖表。
10. caption ＝簡短名詞短語（圖裡有什麼）；意義解讀寫進正文。
11. 主詞全文統一「本研究／本論文」第三人稱（誌謝才用「我」）；「這／它／此」指代不清→「該＋名詞」。

## §1 讀者定位與知識分層

### 1.1 讀者定位
- 核心讀者（正文為其最佳化）：領域同儕。禁教科書式／告知式語氣；每個論點留可被討論的空間，不直接斷言。
- 讀者常為同儕層次、但領域局部（跨領域研究尤然，每位只精通其中一塊）。科普＝跨領域詞彙橋接，非降水準：類比＋一句直白定義＋簡圖，不展開推導（推導指向原始文獻）。
- 簡化要向上可辯護：簡圖可略細節，但底層須對應真實理論、掛得上 citation。正文給簡化直覺、footnote 給精確修正。

### 1.2 知識分層
| 層 | 放什麼 | 判準 |
|---|---|---|
| 正文 | 只放支撐本研究論點的論證主線；用字最精簡 | 「刪掉它，論證會不會垮？」不會→刪或下沉 |
| 文獻回顧章 | 脈絡觀點如何推進到本研究缺口（gap）＋最核心幾篇文獻 | 「不懂它讀不下去本研究」＋「它推進論點」 |
| 附錄 | 完整前情提要：體系背景＋作者前期研究＋複現細節 | 「想補課或想複現才需要」 |
| footnote | 一兩句知識碎片補充，絕不能多 | 「放正文會打斷，不講又不安心」 |
| 原始文獻 | 零複述，只 `\citep`／`\citet` 指路 | 「只是出處」 |

正文遇體系背景→指引附錄／文獻回顧章；遇完整推導→指引原始文獻。

### 1.3 章節職責切割
- 文獻回顧＝argumentative：回答「背景為何把我們逼到研究缺口」，不複述附錄（附錄才是 expository 說明性）。同一主題：附錄講「是什麼」，文獻回顧講「它啟發／推進了什麼」。
- 附錄兩群別混：A＝背景前情提要（體系背景＋作者前期研究）；B＝複現細節（演算法偽碼、參數全表、資料處理程序、資料格式）。
- 章前言 vs 第一節去重：章前言＝行政導航（研究問題承接、章節架構、細節位置），禁劇透核心技術判斷；第一節＝核心判斷首度曝光＋命題正式提出，禁重複前言導航。

## §2 論證與段落

### 2.1 主動宣稱，不被動陳述
每段第一句是可被反駁的論點，非中立觀察。
- 禁：「近年關於 X 的研究越來越多，各種方法被提出。」
- 用：「既有 X 方法共享一個未被質疑的前提：……」

### 2.2 約束驅動鏈式遞進，非組件拆解
禁把最終方案當既定事實逐一介紹元件。要每一層／每個技術選擇都由前一層的結構性限制或現實硬瓶頸逼出。
- 段落句型：`[指出問題／約束] → [現有方法為何失敗] → [宣告本研究決策為邏輯必然]`。
- 段首句必須是約束或衝突。「本研究採用 X」移到段尾；「若不採用 X 則 Y 失敗」移到段首。

### 2.3 動態因果，非靜態報告
- 禁：「A 模組負責產生資料，接下來 B 模組拿去處理。B 模組包含多個子單元…」
- 用：「為解決 [某瓶頸]，A 模組先將 [輸入] 轉換為 [中間表徵]；此表徵進一步制約了 B 模組的設計選擇……」
- 自查：出現「因此／為解決／此限制迫使／唯一化了／排除了」＝論證在推進；大量「負責／包含／由…組成」＝滑入說明書，需重寫。

### 2.4 段落四步結構
`論點句 → 解釋 → 具體說明／證據 → 連結下一段`。
- 開頭：現象／背景先行再收斂主題句（文獻回顧段可引用先行）。
- 結尾：必得結論或橋接下段，不可在引用後直接結束段落。
- 不用「首先…其次…最後…」序數詞串接段落；每點各自用論點句開頭。

### 2.5 小節首尾紀律與去重
- 小節導言＝宣告「本節為解決前節留下的什麼瓶頸而登場」，不可直接寫定義／元件清單。
- 小節結論＝「達成 A，但暴露 B，B 下節解決」，不可只複製本節摘要。
- 一概念一 owner：論點只在擁有它的小節寫完整，他處以名詞帶過。禁下節開頭整段複述前節結論。
- backward reference 刪：參照指向後文未讀者（forward）才留；指向已讀前節者刪。圖／表／式的參照不在此限（讀者需定位）。
- 小結＝新結論：各節結論一句＋指回，再下一個整合性新判斷，非逐節複習。

### 2.6 宣稱三段紀律
每個核心宣稱拆三層、明文標記，不可互相借證據：
| 層次 | 標記 | 說明 |
|---|---|---|
| 動機假說 | 標為本研究立場，引文獻為類比非證明 | 尚未驗證的出發點 |
| 可驗證命題 | 由研究問題／實驗承擔 | 本研究實際檢驗者 |
| 邊界條款 | 明文適用前提 | 成立範圍與限制 |

未驗證的效果標為假說，明文「不在本論文驗證」，不可當已證結果書寫。

## §3 句子與用詞

### 3.1 句構
- 研究動機句型：`[現況／現象鋪陳] → 然而，[具體限制（含引用）] → 因此／有鑑於此，[本研究切入點]`。
- 因果連接詞分工：「因此，」一般邏輯結果／「對此，」針對特定問題某研究如何回應／「基於此結果，」以自己實驗結果推進下一步。
- 概念定義句型：`中文全稱（English, ABBR）為 [簡短功能定義]。具體而言，[舉例／展開]`。
- 一句一核心命題；長句（>60–80 字或含 3+ 括號）拆短。一條因果鏈最多兩層。
- 忌：段落開頭直接「XXX et al. (2022) 提出…」（改現象先、引用後）；單句塞多子句＋術語定義；bullet 動詞不平行；用 bullet list 取代因果推演段落。

### 3.2 用詞與語氣
- 之 vs 的：正式名詞複合鏈用「之」；解釋性修飾用「的」。
- 語氣詞替換：也→亦、都→均、很大→顯著、句首「且」→「此外，」。口語連接詞「再來／接下來／比如」→「其次／此外／例如」。台式「做設計／做比較」→「進行／加以」或直接動詞化。
- 謙遜詞「試圖／希望／期望／盼」每段最多一次；動機句改「本研究提出／建立／驗證」。
- 斷言強度分層：精確可測數據→明確斷言；機制性因果→保留彈性（推測／可能受…影響／傾向）。未驗證用「顯示」不用「證明／顯著」。
- 範圍界定句型：「本研究並非要 A，而是要 B」；對舉句若被否定的 A 不帶資訊，直接寫 B。
- 一個判斷只下一次，避免段尾同義反覆。

### 3.3 術語與中英夾雜
- 首見格式：`中文全稱（English Full Name, ABBR）`；其後全文只用縮寫或中文簡稱，不再重複雙語，也不在後續章節重新定義。
- 同一概念全章固定一詞一寫法。新增術語先比對專案詞彙表。
- 人名保留英文原名不音譯（中文文獻用中文全名）。軟體／工具／演算法／系統名保留英文原文，直接嵌中文句，不加引號斜體。
- 英文動詞化嵌中文語法→改「透過〈英文名詞〉來〈中文動詞〉」。

### 3.4 轉場詞庫與避免用語
| 功能 | 推薦 |
|---|---|
| 引入問題 | 然而／但這個預設…／上述方法的共同盲點在於… |
| 提出論點 | 本研究提出…／核心論點是… |
| 解釋原因 | 這是因為…／其根本原因在於… |
| 給出證據 | 具體而言…／以…為例…／在實驗中… |
| 設定範圍 | 值得注意的是…／本研究並非追求… |
| 轉折限制 | 儘管如此…／這個優勢有其邊界：… |
| 連結下節 | 這引出下一個問題：…／§X 將說明… |

| 避免 | 改用 |
|---|---|
| 眾所周知／顯而易見 | 直接說論點，讓讀者判斷 |
| 非常／十分／極其 | 具體數字或比較 |
| 本文嘗試…／希望能… | 本研究提出／驗證… |
| 如前所述（無指向） | 具體節號或直接重述事實 |
| 清單式「貢獻是…（列三點）」 | 把每條「為何重要」說清楚 |
| 致命／革命性／徹底／龐大／巨大／顯然／根本／最具…特點 | 刪，或改具體數字、比較 |

連續段落禁用同一轉折詞開頭（「也因此／此外／然而」反覆），建同義轉場詞庫輪替。

## §4 標點・數字・格式一致性

### 4.1 標點風格〔專案風格〕
- 標點：中文全形（，。；：「」）；英數字、專有名詞、年代半形。全形（）與半形 () 不混用。
- 破折號「——」：依專案風格設定用量（部分專案規定禁用，改冒號、逗號、連接詞或斷句）；無論用或不用，全文一致。

### 4.2 大陸用語 → 台灣用語（定稿前全文搜尋替換）
| 大陸 | 台灣 | | 大陸 | 台灣 |
|---|---|---|---|---|
| 信息 | 訊息 | | 算法 | 演算法 |
| 缺省／默認 | 預設 | | 內存 | 記憶體 |
| 軟件 | 軟體 | | 硬件 | 硬體 |
| 接口 | 介面 | | 文件夾 | 資料夾 |
| 文件(file) | 檔案 | | 數組 | 陣列 |
| 對象(object) | 物件 | | 集成 | 整合 |
| 通過X | 透過X | | 視頻 | 影片 |
| 屏幕 | 螢幕 | | 字符 | 字元 |
| 加載 | 載入 | | 分佈 | 分布 |
| 質量(quality) | 品質（物理 mass 才用質量） | | 數據 | 資料(data)／數據(數值) |

### 4.3 數字・單位
- 數字：度量衡、年代、圖表號、百分比用阿拉伯數字；中文序數（三個、四種）用國字。句首不以阿拉伯數字起頭（「3 個因素」→「共有三個因素」）。
- 範圍符號擇一全文統一（「a 至 b」或「a～b」，勿混 ~／-／至／～）。
- 單位：同量綱固定一種單位詞；數字與單位間空格規則全文一致。
- 條列：分類用中文數字（一、二、三）、步驟用編號（1. 2. 3.）、並列用項目符號；層級對應全文固定。各項詞性平行、結尾標點一致。

### 4.4 章節與交叉引用〔專案風格〕
- 章的稱呼（中文數字「第三章」或阿拉伯「第 3 章」）依專案樣式，全文統一；改動後對照實際 `\input`／章序確認章號正確。
- 節以下（X.Y、X.Y.Z）與圖／表／式一律用可自動編號的交叉引用（`\ref`），不手寫死號。

## §5 圖表與圖說

### 5.1 圖文分工
- caption ＝簡短名詞短語，只命名「圖裡有什麼」，句末多不加句號。禁寫成完整敘述句、禁含分號／冒號後的詳細說明、禁含解讀句。
- 正文承載意義：在引用該圖的同段或上一段，說清「這圖顯示什麼、意義是什麼」。讀者不看圖也能從正文理解論點。
- 架構／流程圖例外：caption 可詳盡到「圖＋caption 即可重建流程」；但價值判斷／意義一律入正文。
- 自查：正文出現連續「首先…接著…最後…」描述流程傳遞→移入 caption，正文改寫為背後必然性。

### 5.2 圖表義務
- 無孤兒圖表：每個圖表在其出現前，正文先以「（圖 N）／如圖 N 所示」引出並解讀。
- caption 位置：圖說在圖下方、表說在表上方。多子圖用 (a)(b)(c)，caption 逐一對應。
- 非自製圖必標來源（另起一行「（資料來源：…）」）；圖片來源與正式文獻分開列。
- 編號制：全書連續編號 或 章序.流水號，擇一全文統一。
- 數據有效位數全表一致，不超過量測精度；表格標 n、單位、必要誤差／顯著性註記。
- 關鍵論述、caption、詞彙表勿以圖片化文字呈現（PDF 無法檢索）。

## §6 引用與學術誠信

### 6.1 引用格式
- APA 作者—年份制 `（Author, Year）`；`\citep{key}` parenthetical／`\citet{key}` in-text（作主詞帶出論點）。少數專案採腳註制／編號制，全文統一即可。
- 多筆並列同括號內分號分隔。中英文獻混排同一清單（英文在前依字母、中文在後依筆畫）。
- 新文獻加參考文獻檔；引用鍵不捏造，不確定是否存在先列待補。

### 6.2 學術誠信（不可違反）
- 直接引用加「」＋頁碼；改寫仍須標（作者, 年）。禁照抄原文只改幾字當自己的話。
- 引用必須真支持論述，禁掛靠式／堆砌式引用。事實性陳述配 in-text citation，不可只把來源藏圖註。
- 二手轉引先查原文；自我抄襲（搬入自己前期發表）一樣要標註。
- AI 生成／翻譯內容須自行查證，依系所規定揭露 AI 協作。

## §7 全文結構慣例

- 五章制：緒論／文獻回顧／方法與設計／實作與驗證／結論（內容量大可擴六章）。節次三層 `X.Y.Z` 為主。
- 每章章首導言段（無編號）＋章末小結（收束＋橋接下章）。
- 緒論四件套：研究動機 → 研究問題 → 研究目標 → 研究流程與架構（問題／目標編號條列，末節附全文架構圖）。
- 結論四段式：研究回顧／成果 → 貢獻 → 限制（＋改善方向）→ 未來發展，各自獨立成節。研究限制採「限制描述 → 原因 → 改善方向」，禁空泛「仍有改善空間」。
- 頁序：摘要 → Abstract → 誌謝 → 目錄 → 圖目錄 → 表目錄 →（正文）→ 參考文獻 → 附錄。前置頁羅馬數字。
- 摘要結構：背景／現象 → 缺口（然而…）→ 方法 → 實作 →（結果）→ 貢獻／展望。量化型結果段放具體數字。中英一一對應非逐字譯。關鍵詞末行 4–6 個（中文頓號、英文逗號）。中文摘要現在式；英文背景現在式、已執行方法／驗證過去式。
- 敘事主詞全文「本研究／本論文」第三人稱，不用「我」（誌謝例外）。

## §8 自查清單

論證層
- [ ] 每段第一句是現象／論點／約束，還是某某人說／元件宣告？
- [ ] 有無說教／後設句（本節將…／須先言明／正確的第一步…／讀圖時可循…）？
